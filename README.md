# Terminal & Claude Code Drills

A flashcard-style quiz game that teaches you terminal commands and Claude Code — through practice, not memorization.

No installation needed. Just one HTML file.

---

## How to open it

Double-click `index.html`. It opens in your browser like a website.

That's it. Nothing to install, no account needed.

---

## What's inside

**6 categories of questions:**

| Category | What you'll learn |
|---|---|
| Daily Workflow | How to use Claude Code day-to-day |
| Where Am I? | Navigation: `pwd`, `ls`, `cd` |
| Files & Folders | `mkdir`, `touch`, `rm`, `cp`, `mv` |
| Git Essentials | `git add`, `commit`, `push`, `pull` |
| Claude Code | Slash commands, keyboard shortcuts |
| What Went Wrong? | Reading and fixing error messages |
| Speed Tricks | Keyboard shortcuts to work faster |

---

## How it works

Each question comes in one of 4 formats:

- **Type it** — you type the command from memory
- **Pick one** — multiple choice, 4 options
- **What does it do?** — given a command, pick what it does
- **True / False** — is this description correct?

If you get a question wrong, it comes back later in a different format until you get it right.

**Difficulty levels** change the mix:
- Beginner — mostly multiple choice, hints always shown
- Intermediate — mixed formats, hints on click
- Advanced — mostly type-from-memory, no hints

---

## Special modes

**Daily Drill** — 5 questions pulled from your weakest areas. Your scores are remembered between sessions.

**Sandbox** — A fake terminal where you can practice commands freely. Nothing is real, so you can't break anything. Supports `ls`, `cd`, `mkdir`, `touch`, `rm`, `cat`, `git`, `claude`, and more — including Tab autocomplete and arrow-key history.

---

## Keyboard shortcuts (during a drill)

| Key | Action |
|---|---|
| `1` `2` `3` `4` | Pick an answer in multiple choice |
| `T` / `F` | Answer True / False |
| `Enter` | Confirm / move to next question |
| `A` `B` `C` `D` | Also works for multiple choice |

## How to share it

Just send the `index.html` file — by email, AirDrop, Slack, whatever. The other person opens it the same way: double-click.

All progress is saved locally in their browser (no account, no server).
